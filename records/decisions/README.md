# Decisions

Durable project, architecture, workflow, or policy decisions belong here.

## Naming

Use filenames shaped like:

- `DEC-YYYYMMDD-NNN-short-title.md`

Each decision should include:

- `Opened: YYYY-MM-DD HH-mm-ss KST`
- `Recorded by agent: <agent-id>`

## What Belongs Here

- Accepted workflow changes
- Product or architecture choices with rationale
- Policy decisions that future contributors need to understand

## What Does Not Belong Here

- Raw execution history
- Unaccepted brainstorming
- Pure status updates without a decision
