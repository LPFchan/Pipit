# Agent Worklogs

Execution history for migrations, implementation sessions, and notable runs belongs here.

## Naming

Use filenames shaped like:

- `LOG-YYYYMMDD-NNN-short-title.md`

Each worklog should include:

- `Opened: YYYY-MM-DD HH-mm-ss KST`
- `Recorded by agent: <agent-id>`

## What Belongs Here

- Migrations
- Implementation summaries
- Validation sessions
- Session-level notes that explain what actually happened

## What Does Not Belong Here

- Durable product truth
- Accepted future plans
- Decision rationale that belongs in `records/decisions/`
