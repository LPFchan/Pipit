# Pipit — Ninebot G30 BLE Immobilizer Companion App

**Pipit** is the mobile companion app in the [Immogen](https://github.com/LPFchan/Immogen) immobilizer ecosystem for the Ninebot Max G30. It provides background proximity unlock, a foreground fob-style lock or unlock experience, and secure multi-slot key management without requiring a cloud service.

Pipit connects to the **Guillemot** deck receiver and acts as a software companion or alternative to the **Uguisu** hardware key fob.

## Features

1. **Proximity Unlock (Background):** A low-power BLE service detects the vehicle and automatically unlocks or locks it based on configurable RSSI thresholds.
2. **Active Key Fob (Foreground):** A single-screen utility surface presents an interactive 3D fob experience for manual unlock and lock actions.
3. **Multi-Slot Key Management:** Pipit manages up to four firmware-defined slots, including owner and guest devices, migration, rename, revoke, and recovery flows.
4. **Offline Provisioning:** Owner setup uses Argon2id-encrypted QR payloads generated by [Whimbrel](https://github.com/LPFchan/Whimbrel), while guest provisioning can remain plaintext.

## Tech Stack

- **Shared Core:** Kotlin Multiplatform handles AES-CCM crypto, Argon2id-backed provisioning helpers, monotonic counters, onboarding gates, QR parsing, and shared settings keys.
- **Android:** Jetpack Compose plus the platform BLE stack, a foreground service, SceneView-based 3D presentation, and a USB OTG backend.
- **iOS:** SwiftUI plus platform BLE APIs and a bundled WebKit-backed Three.js viewer for the fob surface.
- **Background Services:** CoreLocation plus CoreBluetooth on iOS, and foreground-service BLE scanning on Android.

## Build Requirements

- **Java:** Use JDK 17 to run Gradle in this repository.
- **Gradle Wrapper:** The project uses Gradle `8.13`.
- **Android Gradle Plugin:** Android modules are aligned on AGP `8.13.2`.
- **Android SDK:** Android builds require platform 35 via `local.properties`, `ANDROID_HOME`, or `ANDROID_SDK_ROOT`.

## Commit Provenance Enforcement

Pipit enforces commit provenance both locally and in CI:

- Run `scripts/install-hooks.sh` once per clone to configure `core.hooksPath` to `.githooks`.
- Local commits then pass through `.githooks/commit-msg`, which runs `scripts/check-commit-standards.sh`.
- `.github/workflows/commit-standards.yml` validates pushed and pull-request commit ranges remotely.
- A normal commit must reference at least one relevant artifact, but it can update an existing `LOG-*`, `DEC-*`, or `RSH-*` instead of creating a new worklog by default.

## Ecosystem Integration

Pipit interoperates with other Immogen projects through shared protocol constraints:

- **Immogen / Guillemot firmware:** Owns slot semantics, management commands, and BLE payload validation.
- **Whimbrel dashboard:** Generates provisioning QR payloads that Pipit must parse and decrypt correctly.
- **Bifrost sync workflow:** This repo can trigger Bifrost on push through `.github/workflows/trigger-bifrost-sync.yml`.
